# 1. ТЕОРЕТИЧНИЙ ФУНДАМЕНТ

## 1.1 Дефініція програмного пакета
**Пакет** — це спеціалізований формат архіву, адаптований для автоматизованого розгортання в ОС Linux. Він інкапсулює в собі:
* Компіляційні бінарні дані та системні бібліотеки;
* Дерево інструкцій щодо задоволення необхідних залежностей;
* Конфігураційні пресети та скрипти пост-інсталяції;
* Метадані (версія ПЗ, авторство, контрольні суми для перевірки цілісності).

Для екосистеми Ubuntu/Debian базовим стандартом є формат **.deb**.

## 1.2 Концепція репозиторію
**Репозиторій** — це віддалений мережевий вузол (серверне сховище), що виконує роль централізованого банку програмного забезпечення. Використання репозиторіїв дозволяє підтримувати безпеку та стабільність системи через використання цифрових підписів та автоматизацію оновлень.

## 1.3 Глосарій професійної термінології

| Термін | Технічна інтерпретація |
| :--- | :--- |
| **Package** | Одиниця програмного забезпечення, що містить код та інструкції для інсталяції. |
| **Repository** | Джерело дистрибуції, де пакети індексуються та зберігаються за версіями. |
| **Package Manager** | Системна утиліта, що контролює встановлення, оновлення та видалення ПЗ. |
| **APT Interface** | Високорівневий консольний інструмент для керування пакунками (Advanced Package Tool). |
| **Dependency Tree** | Сукупність додаткових компонентів, необхідних для коректного запуску програми. |
| **System Upgrade** | Процес заміни встановленого софту на новіші ітерації з виправленнями. |
| **Purge Method** | Радикальне видалення програми разом із її глобальними конфігураціями. |
| **Snap Ecosystem** | Універсальні контейнеризовані застосунки, ізольовані від ядра системи. |
| **Flatpak Stack** | Крос-платформенна технологія розгортання ПЗ у захищеній "пісочниці". |
| **CLI Environment** | Взаємодія з операційною системою через текстові команди (Terminal). |
| **PPA (Personal)** | Персональні архіви пакетів для отримання ПЗ поза офіційною базою Ubuntu. |

## 🛠 1.4 Порівняльний аналіз менеджерів пакетів

Кожна гілка дистрибутивів Linux базується на власних методах пакування коду.

| Дистрибутив | Утиліта | Формат | Характерні риси |
| :--- | :--- | :--- | :--- |
| **Ubuntu / Debian** | `apt` | `.deb` | Висока стабільність, гнучка робота із залежностями. |
| **Fedora / RedHat** | `dnf` | `.rpm` | Швидка обробка метаданих, підтримка транзакцій. |
| **Arch Linux** | `pacman` | `.zst` | Орієнтація на Rolling Release (постійні оновлення). |
| **openSUSE** | `zypper` | `.rpm` | Потужна інтеграція з інструментами налаштування YaST. |
| **Snap / Flatpak** | `snap/flatpak` | `atomic` | Повна незалежність від версій системних бібліотек. |

# 2. ПРАКТИЧНИЙ ПРОТОКОЛ РОБОТИ З APT

## 2.1 Оновлення індексів ПЗ
Команда ініціює синхронізацію локальної бази даних із серверами репозиторіїв для актуалізації списків доступного софту.
```bash
sudo apt update
🔎 Переглянути скріншот

2.2 Верифікація наявності пакета (Пошук)
Використовується для знаходження точної назви додатка та доступних версій у підключених джерелах.

Bash
apt search vlc

2.3 Ініціалізація встановлення (VLC)
Завантаження та автоматичне розгортання медіаплеєра VLC разом із усіма необхідними кодеками та бібліотеками.

Bash
sudo apt install vlc
2.4 Аналіз метаданих (Show)
Отримання повної технічної інформації про версію, залежності, опис та розмір інсталяційного пакета.

Bash
apt show vlc
2.5 Ревізія встановленого софту
Перегляд повного переліку програм та бібліотек, які наразі інтегровані в операційну систему.

Bash
apt list --installed
2.6 Процедура деінсталяції
Видалення виконуваних файлів програми при збереженні користувацьких конфігурацій для можливого майбутнього використання.

Bash
sudo apt remove vlc

2.7 Санація системи (Autoremove)
Очищення ОС від транзитивних бібліотек, які стали непотрібними після деінсталяції основного програмного забезпечення.

Bash
sudo apt autoremove

2.8 Глобальне оновлення пакетної бази
Комплексне оновлення всіх встановлених пакетів до найактуальніших стабільних версій, доступних у репозиторіях.

Bash
sudo apt upgrade

3. РОЗГОРТАННЯ СЕРЕДОВИЩА РОЗРОБКИ
Екосистема Python
Інсталяція інтерпретатора Python 3 та офіційного графічного середовища розробки IDLE для написання скриптів.

Bash
sudo apt install python3 idle3

Інструментарій C/C++ (build-essential)
Встановлення метапакета, що містить компілятор GCC, g++, утиліту make та базовий набір системних бібліотек.

Bash
sudo apt install build-essential

Проходження процедури автентифікації через пароль суперкористувача.
